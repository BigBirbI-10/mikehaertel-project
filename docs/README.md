## Documentation Standards

### File Naming
- Use lowercase with hyphens: `aws-setup.md`
- Be descriptive but concise
- Group related docs with prefixes when needed

### Content Structure
Each document should include:
1. Title and purpose
2. Last updated date
3. Table of contents (for longer docs)
4. Clear sections with headers
5. Code examples where applicable
6. Links to related documentation

### Keeping Docs Current
- Update documentation when code changes
- Note the last updated date
- Reference the related changelog entry
- Review quarterly for accuracy

## Contributing to Documentation

When adding or updating documentation:
1. Follow the established format
2. Update this index if adding new files
3. Cross-reference related documents
4. Include practical examples
5. Keep language clear and concise

## Questions?

If documentation is unclear or missing, create an issue or contact the project maintainer.
